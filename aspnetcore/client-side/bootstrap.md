### <a name="grids"></a>グリッド

ブートストラップの最も一般的な機能の 1 つは、そのグリッド レイアウト システムです。 最新の web アプリケーションを使用しないように、`<table>`レイアウト、代わりにこの要素の使用を制限する実際の表形式データへのタグ。 代わりに、列と行を配置できる一連の`<div>`要素および適切な CSS クラス。 これにはグリッドに表示する垂直方向に狭い画面など、携帯電話でのレイアウトを調整する機能など、この方法にいくつかの利点があります。

[ブートストラップのグリッド レイアウト システム](http://getbootstrap.com/css/#grid)12 個の列に基づきます。 1、2、3、または 4 つの列に均等に分割できますされ、列の幅を 1 内に変更できるため、この数は選択された画面の垂直方向の幅の 12/です。 グリッド レイアウト システムを使用するには、コンテナーで開始する必要があります`<div>`行を追加および`<div>`次に示すように、します。

```html
<div class="container">
    <div class="row">
        ...
    </div>
</div>
```

次に、さらに追加`<div>`要素、各列の列の数を指定する`<div>`"col-md-"で始まる CSS クラスの一部として (12) 外を占有する必要があります。 たとえば、単に等しいサイズの 2 つの列がある場合は、それぞれの「col md 6」のクラスを使用するは。 ここで"md"は"medium"の略でと、デスクトップ コンピューターの標準サイズの表示サイズを参照します。 選択することができます、4 つの異なるオプションがあるし、(画面の幅に関係なく修正するレイアウトを設定する場合は、xs クラスだけ指定できます) ためにオーバーライドされない限りに高い幅それぞれ使用します。

CSS クラスのプレフィックス | デバイスの階層 | 幅
:---: | :---: | :---:
col-xs- | 携帯電話 | < 768px
col-sm- | タブレット | > = 768px
col-md- | デスクトップ | > = 992px
col-lg- | 大規模なデスクトップの表示 | > = 1200px

2 つの列両方「col-md-6」、結果のレイアウトでは解像度がデスクトップの 2 つの列になりますが、上での小型デバイス (デスクトップの幅の狭いブラウザーのウィンドウ)、ユーザーが簡単に表示できるようにレンダリングされるときにこれら 2 つの列が垂直方向にスタックを指定する場合コンテンツ水平方向にスクロールする必要はありません。

ブートストラップは常に既定値 1 列のレイアウトでは、のみ 1 つ以上の列を表示するときに列を指定する必要があります。 明示的に指定するだけの時間、 `<div>` 12 のすべての列をより大きなデバイス層の動作をオーバーライドすることです。 複数のデバイス層クラスを指定する場合は、特定の時点で列の描画をリセットする必要があります。 特定のビューポート内に表示される clearfix div を追加することを実現できますこれには、次に示すように。

![ナローとワイドのビューポートのグリッド](bootstrap/_static/narrow-and-wide-viewport-grid.png)

上記の例では、1 と 2 つ共有"md"レイアウト内の行 2 および 3 は"xs"レイアウト内の行を共有します。 なし、clearfix `<div>`2 と 3 では、その"xs"ビュー (1 つだけ、4、および 5 が示されているメモ) では正しく表示されないは。

![clearfix を使用せずにグリッド](bootstrap/_static/grid-without-clearfix.png)
